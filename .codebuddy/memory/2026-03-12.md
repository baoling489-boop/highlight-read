## 修复三个问题
- **翻页分栏**（已二次修复）：最初注入 `column-count:1 !important` 来防止分栏，但这破坏了 epub.js 的分页机制（epub.js 的翻页模式正是靠 CSS columns 实现分页的），导致翻页模式下 prev()/next() 直接跳到上/下一章而非翻页。二次修复：移除所有 column 相关的 CSS 覆盖，让 epub.js 自然通过 columns 分页，`spread:'none'` 已经确保单栏。
- **页边距/行间距不生效**：epub.js 的 `themes.override` 设置 `!important` 不够可靠。改为全部通过 `injectCustomStyles` 直接注入 iframe CSS，用 `useLatest` ref 读取最新值，参数变化时实时更新。
- **全书进度条**：原来是章节内进度（displayed.page / displayed.total）。改为 `book.locations.generate(1024)` 生成全书位置索引，用 `book.locations.percentageFromCfi()` 计算全书百分比。

## 页边距最终方案：固定外层 margin + 热区合一
- **问题**：之前多次尝试通过修改 epub.js iframe 内部的 body padding 来实现页边距，但 epub.js 翻页模式的 `columns()` 方法会通过 inline `!important` 覆盖 padding，且修改后 column-width 不同步导致双栏。
- **最终方案**：彻底放弃修改 epub.js 内部 padding。改为在**外层布局**实现：
  - 定义常量 `FIXED_PAGE_MARGIN = 60`（px）
  - `readerArea` 用 `display: flex; flex-direction: row` 横向布局
  - 左右各放一个固定宽度（60px）的 div，既是页边距也是翻页热区
  - epub.js 的 `viewer` 容器在中间，宽度 = 容器宽度 - 120px
  - 移除了 `pagePadding` state、`applyPagePadding()` 函数、Toolbar 中的页边距调节控件
  - 热区不再是 absolute 定位覆盖在文本上，而是 flex 子元素占据固定宽度

## 翻页模式内容贴边：gap: 0
- **问题**：翻页模式下 epub.js 内部会自动计算 gap（`Math.floor(width/12)` 取偶数），然后设 `padding-left: gap/2` 和 `padding-right: gap/2`，导致内容没有铺满 viewer 区域，实际页边距 = 外层 60px 热区 + epub.js 内部 padding，视觉上不贴边。
- **根因**：epub.js `layout.calculate()` 中，如果未传入 `_gap` 参数（`!(_gap >= 0)` 为 true），会自动计算 gap。
- **修复**：在 `renderTo` 选项中加 `gap: 0`，这样 `_gap >= 0` 条件为 true，跳过自动计算，gap 保持为 0，epub.js 内部不设水平 padding。
- **同时**：在 `injectCustomStyles` 中加了 `padding-top: 20px` 和 `padding-bottom: 20px` 保持上下间距（因为 epub.js columns() 设的 padding-top/bottom 也是 20px，但设 gap:0 后不确定是否还生效）。

## 进度计算改为基于字数
- **问题**：之前使用 `book.locations.percentageFromCfi()` 计算进度，只在 `relocated` 事件（章节切换）时触发，滚动阅读时进度不会实时更新
- **新方案**：
  - 在 `book.ready` 后遍历所有 spine items，用 `book.load()` 加载每章内容，统计去空格后的字数
  - 存储每章字数和累计字数到 `chapterCharsRef`，总字数到 `totalCharsRef`
  - 在 iframe 的 `scroll` 事件中，用 `scrollTop / maxScroll` 得到章节内滚动比例
  - 全书进度 = (前面章节累计字数 + 当前章节字数 × 滚动比例) / 总字数
  - 章节切换时（`relocated`）以滚动比例 0 重新计算
- **修改文件**：`EpubReader.tsx`
